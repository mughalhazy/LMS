runtime_operation | input_data | result
--- | --- | ---
SCORM package launch | `learner_id`, `course_id`, `registration_id`, SCORM package manifest (`imsmanifest.xml`), selected SCO identifier, launch mode (`normal`/`review`/`browse`) | LMS resolves SCO launch URL, initializes SCORM API session, creates runtime attempt record, and returns launch context with session token and initial CMI values.
Progress tracking | Runtime API events from SCO (`LMSSetValue`/`SetValue`, `lesson_location`, `suspend_data`, `session_time`, `score`, interactions/objectives), commit events (`LMSCommit`/`Commit`) | LMS validates and persists CMI updates, checkpoints learner state for resume, updates progress percentage and time spent, and stores attempt history for analytics.
Completion reporting | Final runtime signals (`LMSFinish`/`Terminate`), completion fields (`lesson_status` or `completion_status`/`success_status`), mastery score, total time, attempt limits/policy rules | LMS computes final attempt outcome (completed/passed/failed/incomplete), records completion timestamp and score, updates course enrollment status, and emits completion event/report payload to downstream systems.
