Part 1 · Foundations
10 Design Axioms
These are laws. They do not bend unless they demonstrably break usability, conflict with accessibility requirements, or prevent the system from serving its users. Evolution requires documented rationale + version bump.
1
Clarity is King
Design and function both serve clarity. Not the other way around.
2
Light = Clarity
Cognitive load lives in darkness and density. Keep it warm and open.
3
Semantic Color
Every color has exactly one meaning, always. Color that appears everywhere signals nothing.
4
1.5px Borders on Emphasis
1px is furniture. 1.5px is a signal. The difference is intentional and enforced.
5
Warm Off-White Canvas
#F7F7F5 — not cold grey. Reduces eye strain in learning contexts.
6
Color Has Four States
dk (text) · md (fill/accent) · lt (background tint) · b (border rgba)
7
Repetition Builds Trust
The same pattern every time creates user fluency. Variation is a tax.
8
No Dark Panels
Killed in v1. Never revisited. The Admin surface uses ink on white.
9
80 / 12 / 8 Rule
80% neutral · 12% primary accent (indigo) · 8% semantic signals. Count > 3 colors simultaneously → too many.
10
Color on the Indicator Only
Color lives on dots, stripes, fills, numbers. Never on containers.

Part 2 · Interaction Language
Interaction Rules
Predictable behavior across the entire product. Every rule exists because an alternative was tried and created confusion.
One primary action per page
Primary button (ink fill) appears once per page, right-aligned in the page header. Secondary actions: max 3, ghost or outlined.
Destructive actions always confirm
Modal must name the specific entity ("Delete Alex Jordan?" not "Delete user?") and state if irreversible. Brick color on confirm button only.
Row actions surface on hover only
Max 2 icon buttons + ellipsis overflow. Never permanently visible — reduces visual noise in dense tables.
Skeleton before spinner — always
Skeleton screens preserve layout and reduce perceived load time. Full-page spinners are never used. Skeleton mirrors loaded layout exactly — same dimensions, same positions.
Never wrap table cells
All table cells truncate with single-line ellipsis. Wrapping breaks row height consistency and the Admin density contract.
Never use color on containers
The 80/12/8 rule. Color lives on dots, stripes, fills, numbers. Colored card backgrounds were tried and removed in v1 — they were decoration, not signal.
Never show an empty table with 0 rows
Always use the EmptyState component. Resource-specific illustration, specific headline, single CTA. Generic "no data" messages violate the tone contract.
