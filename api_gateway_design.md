gateway_component | responsibility | target_services

Edge API Gateway (Ingress + TLS terminator) | Central API entry point for all external LMS traffic; enforces HTTPS, CORS, and request normalization before forwarding downstream. | auth-service, user-service, course-service, content-service, assessment-service, progress-service, analytics-service, notification-service, integration-service, admin-service
Route Manager (Path + Method Router) | Routes requests by URL path, HTTP method, version, and tenant context to the correct backend microservice endpoint. | auth-service, user-service, course-service, content-service, assessment-service, progress-service, analytics-service, notification-service, integration-service, admin-service
Authentication & Authorization Filter | Enforces authentication using JWT/OAuth2 token validation, propagates user/tenant claims, and blocks unauthorized access before service execution. | auth-service (token introspection/JWKS), user-service, course-service, content-service, assessment-service, progress-service, analytics-service, notification-service, integration-service, admin-service
Rate Limiting & Throttling Module | Applies per-tenant, per-user, and per-client quotas with burst control to protect services from abuse and noisy-neighbor traffic. | all backend services via gateway policy
Request Logging & Audit Middleware | Captures structured request/response metadata (trace ID, actor, route, latency, status) for observability, compliance, and incident investigation. | observability stack (log-service/SIEM/APM) + all backend services (via trace propagation)
